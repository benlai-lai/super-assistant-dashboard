# Codex 開發任務書：超級隨身助理 Dashboard V1

## 任務目標
依照 `PRODUCT_SPEC.md` 與 `index.html`，建立第一版可操作的前端 Prototype。重點是讓使用者在 30 秒內看懂今日工作狀態，並可在桌機與手機使用。

## 必須完成
- 保留目前 Dashboard 資訊架構。
- 完成總覽、日曆、任務、專案與 Inbox 視圖。
- 完成導覽切換、快速新增、任務狀態切換、完成與封存。
- 快速新增需支援三種類型：想法、任務、行程。
- 使用 `localStorage` 保存示範資料。
- 加入預設資料：超級隨身助理、AI Sales OS、BagKing 網站、台北郵局拜訪。
- 手機版底部導覽可操作。
- 全站使用繁體中文。

## 本階段禁止事項
不得實作：
- Google Calendar 或 Apple Calendar 串接
- 真正語音辨識
- 後端 API 或資料庫
- 登入與多人協作
- 推播提醒
- AI 模型串接
- 未經規格定義的新模組

如有建議，列入交付報告，不得自行擴充。

## 首頁資訊順序
1. 今天最重要的三件事
2. 今日行程
3. 等待使用者決策
4. 專案進度
5. 即將到期與逾期事項
6. 任務狀態摘要
7. 小型月曆

日曆不可成為首頁主體。

## 建議技術
- HTML
- CSS
- Vanilla JavaScript
- localStorage

除非 Repository 已有既定框架，否則不要引入 React、Vue、Next.js 或大型 UI 套件。
